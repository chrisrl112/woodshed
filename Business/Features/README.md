# Features

Feature ideas, prioritization, and roadmap thinking — the "what should we build and why" layer that sits above individual build specs.

Detailed technical/build specs for approved features still live in `docs/` (e.g. `WOODSHED-BACKLOG-*.md`, `*-BRIEF.md`). Put things here when they're still at the idea/prioritization stage, or when you're weighing feature tradeoffs from a business rather than implementation angle.
