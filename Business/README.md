# Business

Non-technical, business-side work on Woodshed — as opposed to `pipelines/`, `charts/`, `vendor/`, etc., which are the product/codebase. If it's a decision, a plan, a piece of positioning, or an AI-run process *about* running the business rather than *inside* the app, it lives here.

| Folder | Scope |
|---|---|
| `Branding/` | Brand strategy, voice/positioning, guidelines. (Served logo/icon files stay in `assets/brand_assets/` — see that folder's README for why.) |
| `Features/` | Feature ideas, prioritization, roadmap thinking — the "what should we build and why" layer above individual specs in `docs/`. |
| `Design/` | UX/visual direction, design decisions and rationale not already captured in `docs/` build specs. |
| `Social/` | Content strategy, reels, community/platform recon (Reddit, etc.), community mapping. |
| `Strategy/` | Business strategy, monetization, sequencing, market positioning. |
| `Agentic/` | Meta-AI-ops: prompt caching, model selection, agent oversight/PM — how AI is used to *run the business*, not the product's own AI features (those live in `pipelines/` and `docs/`). |

Each subfolder has its own README with more specific scope notes and pointers to related material elsewhere in the repo.
